    聊天机器人斜杠指令

    Kesoku 聊天机器人适配器支持标准的斜杠指令（例如在 Discord 中）或文本命令前缀（例如在微信或命令行终端中输入 /command），用于控制智能体执行流、管理上下文压缩、切换人物设定以及监控系统性能。

    🚀 可用指令列表

    1. 人设管理 (Persona)

    • /role：显示当前活跃的人物设定名称，并列出工作区中所有可用的角色列表。
    • /role {名称}：为当前频道或子线程绑定并切换到指定的角色（例如 /role coder）。

    2. 上下文与会话控制

    • /clear（别名：/reset）：
      • 从 SQLite 数据库中永久删除当前频道/子线程的全部聊天历史。
      • 递归删除磁盘上该会话专属的暂存区文件夹（清理所有下载的文件与临时附件）。
      • 清除界面上处于活动状态的按钮视图。
    • /compact：手动强制对当前频道的会话历史进行上下文压缩，将老回合打包为轻量的树状摘要节点，以释放上下文 Token 窗口。
    • /context：生成并发送一个可供下载的交互式暗黑模式 HTML 轨迹文件（lcm_context.html），详细展示当前回合大模型的思考链路（Thoughts）、原始 Prompt 以及工具执行日志。
    • /grep {查询}（别名：/memory-grep）：从当前频道所绑定的角色人设下，通过关键字或通配符 * 检索历史消息和主动记忆。
    • /search {查询}（别名：/memory-search）：利用向量嵌入对当前频道所绑定的角色人设下的历史消息和主动记忆进行语义搜索（Semantic Search）。

    3. 性能监控与调试

    • /status：获取并显示当前会话的详细执行指标：
      • 总会话回合数：当前会话处理过的用户提问次数。
      • 上下文窗口大小：加载入上下文的活跃 Token 数量（以 K 字节为单位）。
      • 已执行工具次数：当前回合调用的工具总数。
      • 单回合 Token 消耗量：输入与输出 Token 的累计消耗量。
      • 耗时与速度：最新回合的实际运行时间（以秒为单位）。
    • /debug：即时切换调试模式。启用后，Kesoku 会将原始 LLM 推理 JSON 写入日志，并返回会话在磁盘上的绝对路径以供审计。

    4. 服务系统

    • /restart：安全地重启 Kesoku 后台守护进程服务。
      • 发送临时消息确认收到重启指令。
      • 清理连接并优雅停止后台监听器。
      • 重新加载 OS systemd 守护进程或执行 execv 原地进程替换。
    • /cronjob（别名：/cronjob {tag}）：列出或手动触发配置文件中注册的定时自动化任务。
